# Functional

Functional spacing variables have semantic meaning and opinionated spacing values. They are meant to cover the most common UI patterns.

We use terminology from [CSS logical properties](https://css-tricks.com/css-logical-properties/) to reference dimensions, where the horizontal axis is represented as inline, and the vertical as block.

## UI Control

UI controls specify common dimensions found in interactive components.

These values are shared by buttons, input fields, tabs, segment controls, toggles, action list items, and more.

UI controls are available in different sizes, and always include a size variant, represented as control-medium, control-large, etc. The size variant represents the expected computed height, when the interactive component has a single line of text.

<Blockquote>
<p><strong>size</strong>: The height of a control, defined in rem. If your component can grow vertically (e.g. have multiple lines), you may set the correct spacing dimensions with <strong>paddingBlock</strong> instead.</p>


<p>
  <strong>paddingBlock</strong>: Sets the vertical padding. Can be used to support interactive components that have
  multiline.
</p>

<p>
  <strong>paddingInline</strong>: The horizontal padding of a control. Available in up to 3 independent sizes:
  `condensed` | `normal` | `spacious`
</p>

<p>
  <strong>lineBoxHeight</strong>: Value in rem with the computed size of the line height, used for component composition
  calculation.
</p>

<p>
  <strong>gap</strong>: The spacing between elements inside a control.
</p>

</Blockquote>
